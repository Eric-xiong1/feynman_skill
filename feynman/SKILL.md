# Feynman

## Overview

把难懂概念翻译成能复述、能迁移、能继续追问的中文直觉。默认服务“读论文、教材、技术文档卡住”的即时解惑场景。

核心原则：先给答案，不先发菜单；先讲当前卡点，不把用户拖进完整课程。

## Automatic Trigger Contract

自动触发 requires both material and signal：用户给了一个需要理解的材料或概念，并且表达了困难、简化、费曼学习或自测意图。只有“解释一下”不够；只有“好难”但没有对象也不够。

| Signal group | Typical cues | Trigger behavior |
| --- | --- | --- |
| Confusion signals | 不懂、看不懂、听不懂、不会、好难、卡住、懵了、还是不明白、这段在讲啥、什么意思、为啥、为什么 | 直接进入默认解惑三件套 |
| Plain-language signals | 简单讲、讲简单点、通俗易懂、说人话、大白话、小白能懂、别用术语、讲给孩子听 | 进入默认解惑三件套，并降低术语密度 |
| Learning-mode signals | 费曼、类比、打个比方、知识树、概念树、先修知识、测测我懂没懂、我来讲你挑刺 | 进入对应路由，必要时加载 reference |

Non-trigger boundaries：

- 用户只是说“解释一下这段代码/这个报错/这个 API 怎么用”，优先按代码、调试或工具文档任务处理，除非同时出现上述困难或通俗化信号。
- 用户只是要求“总结、翻译、改写、润色、写文案”，不要自动套用费曼结构，除非出现“看不懂/说人话/费曼/类比”等信号。
- 用户请求事实查询、最新信息、法律医疗金融建议时，不因为“为什么”自动触发；按原任务的准确性和来源要求处理。

## Default Flow

当输入满足自动触发契约，且材料是概念、术语、论文段落、教材段落、英文技术解释、代码注释、公式说明或理论机制时：

1. 推断用户意图、概念类型、卡点和受众；没有明确受众时，默认“非本专业的成年外行”。
2. 直接输出中文解释；不要要求用户先选择受众、风格或模式。
3. 默认输出“解惑三件套”：通俗演绎、生活类比、黑话翻译。
4. 结尾追加一个理解检验追问，再给一行自然语言换挡邀请。

默认结构：

```markdown
### 一句话抓住它
[用一句人话说清这个概念到底在解决什么问题]

### 人话解释
[分 2-4 段解释核心机制。先讲直觉，再补必要边界]

### 生活类比
[给一个完整类比，并列出 3 组以上局部对应]

### 黑话翻译
- [术语]：[标准意思]。说人话就是：[生活化解释]。

### 反问一句
[一个能暴露理解漏洞的追问]

想换个角度的话，可以说：讲给5岁听 / 用做菜比喻 / 画知识树 / 测测我懂没懂。
```

## Routing

按自然语言意图加载参考文件，避免每次加载全部内容。

| 用户意图 | 行为 |
| --- | --- |
| 指定“讲给5岁/10岁/老人/外行/同行听” | 读取 `references/audiences.md`，按受众校准措辞、抽象层级和例子 |
| 指定“用比喻/故事/对话/做菜/说明书讲” | 读取 `references/styles.md`，按风格重写或继续解释 |
| 要“知识树/概念树/先修知识/结构图/Mermaid” | 读取 `references/concept-tree.md`，生成结构化知识拓扑 |
| 要“测测我懂没懂/我来讲你挑刺/三人格自测” | 读取 `references/personas.md`，进入自测评估 |
| 继续追问“为什么/边界/反例/现实用途” | 直接延续当前解释，必要时读取相关 reference |

## Output Standards

- 使用简体中文，除非用户明确要求其他语言。
- 解释优先贴近用户贴出的原文，不把一个窄卡点扩写成百科全书。
- 保留关键事实准确性；为降低抽象度而简化时，标明“这个比喻只对应 X，不对应 Y”。
- 把公式、符号和英文缩写翻译成普通文本；只有用户要求时才保留复杂数学形式。
- 黑话翻译只挑 2-4 个真正阻碍理解的中介词，不堆术语表。
- 类比必须有局部对应关系，不能只说“它像某某”。
- 用户已经给出理解或解释时，先评价其理解，再补洞。

## Common Mistakes

| Mistake | Correction |
| --- | --- |
| 先问“你想听什么风格” | 直接用智能默认解释，结尾再给换挡邀请 |
| 只把术语换成更口语的术语 | 找出机制、因果和边界，用生活场景重建 |
| 类比只有一句话 | 给出至少 3 个“原概念部件 = 类比部件”的对应 |
| 讲得很萌但牺牲准确性 | 明确比喻边界，补上不会误导的关键限制 |
| 用户要自测时继续讲课 | 转入倾听者/考官模式，让用户先复述或评价其复述 |
